# IOM Module Business Flow — BA Confirmation Draft

## 1. Module Name

IOM — Internal Office Memo

## 2. Purpose

The IOM module will allow CRM, CRM Verifier/TL, CRM Head, Finance, and Loyalty users to process eligible loyalty booking records, generate IOM documents, complete approval/signature workflow, release loyalty points, manage invoice request/submission, and track the full journey through role-wise dashboards.

Super Admin will have read-only visibility of all IOM records for monitoring and audit.

---

# 3. Users / Roles Involved

## 3.1 CRM User / Created By

CRM user will:

- Login through existing SSO + OTP.
- See only project-specific booking records assigned to the CRM.
- Upload signature from profile management.
- View eligible booking records where:
  - Source is Loyalty.
  - Source can be Purva Privilege or Provident Premier.
  - 15% payment has been received.
  - Booking data comes from SAP API.
- Generate IOM from prefilled booking data.
- Submit generated IOM for approval.
- Review generated IOM PDF.
- Retry failed notification if system exposes failure state.

## 3.2 CRM Verifier / CRM TL

CRM Verifier / CRM TL will:

- View IOMs submitted by CRM users.
- Verify IOM.
- Place verifier signature.
- Trigger notification to CRM Head after verification.
- Review generated IOM PDF.
- Retry failed notification if allowed.

## 3.3 CRM Head

CRM Head will:

- View IOMs verified by CRM Verifier/TL.
- Approve IOM.
- Place CRM Head signature.
- Trigger notification to Finance after approval.
- Review generated IOM PDF.
- Retry failed notification if allowed.

## 3.4 Finance User

Finance user will:

- Login through existing SSO + OTP.
- View records approved by CRM Head.
- Review IOM PDF.
- Place Finance signature.
- Close IOM.
- Trigger notification to Loyalty after Finance closure.
- Close or update invoice where applicable.
- Retry failed notification if allowed.

## 3.5 Loyalty User

Loyalty user will:

- Login through existing SSO + OTP.
- View records after Finance approval/closure.
- Fetch customer details from Pinelabs.
- Match IOM customer details with Pinelabs customer details.
- Create a new Pinelabs profile if customer does not exist.
- Mark points as eligible.
- Mark points as redeemable.
- Release loyalty points.
- Push data to Salesforce if backend confirms support.
- Request invoice in bulk.
- Submit invoice details in bulk.
- View daily pool amount if daily pool API is available.
- Retry failed notification if allowed.

## 3.6 Super Admin

Super Admin will:

- View all IOM records.
- Review PDF.
- Export records.
- Not perform operational workflow actions in current scope.

Metrics/dashboard for Super Admin and CRM Head are out of scope for now unless business confirms otherwise.

# 4. Entry Condition for IOM Flow

A booking record becomes eligible for CRM IOM flow when all below conditions are met:

1. Booking source is Loyalty.
2. Loyalty source is Purva Privilege or Provident Premier.
3. 15% of sale value has been received.
4. Booking record is available through SAP API.
5. Booking belongs to the CRM user’s assigned project.
6. No active/approved IOM already exists for that booking record.
